## Session 2026-07-16

- Q: The spec predicts a `POST /api/kudos` + `/api/upload` + Supabase Storage backend — is that shipped? → A: No. The modal is **mock/demo** with **no backend**. Submit builds a `KudosPost` and calls `addKudos()` on a client-side session store (`lib/saa/kudos-store.ts`); the All Kudos list subscribes via `useSyncExternalStore` so the new card appears at the top immediately (not persisted, gone on reload). Images are a fixed `SAMPLE_IMAGE` placeholder — only the **count** (`photos`) is carried, no upload.
- Q: What is the spec's "Frame 552 campaign/award-type selector" (Open Q1)? → A: It ships as the **"Danh hiệu" (title) text field**, not a campaign dropdown. The value becomes `hashtagTitle` on the posted kudos (falling back to `t.titleLabel` when blank). There is no campaign/award-type picker.
- Q: How strict is validation vs. the spec's "disable Gửi until all valid + red borders on recipient/message/hashtag"? → A: Relaxed to **recipient-only required**. An empty "Người nhận" shows a red border + `t.recipientRequired` and re-opens the dropdown; title, body and hashtags fall back to defaults if empty, and "Gửi" is always enabled. Per-field errors on message/hashtag are deferred.
- Q: How is the hashtag five-item cap enforced (Figma disables unselected rows at 5)? → A: The shipped `write-kudos-modal.tsx` instead **hides the "+ Hashtag" add button** (and thus the picker) once `tags.length === MAX_ITEMS` (5). Chips stay removable, so the count can drop below five and the button returns. Same cap, different mechanism. The picker reuses `SaaDropdownPanel`/`SaaDropdownItem` with `checked` + `size="compact"` (gold/20 wash + check badge).
- Q: What about the spec's rich-text toolbar, "@mention", loading/error toasts and focus trap? → A: The toolbar (B/I/S/list/link/quote) and "Tiêu chuẩn cộng đồng" link are **visual-only** glyphs; "@" is a **hint line only** (`t.mentionHint`), no autocomplete. No submit spinner / error toast (there is no async call). Escape + backdrop close and body-scroll lock are wired; a full focus trap is deferred.
- Q: Where is the modal launched from, and how does anonymous send appear? → A: Three entry points, all opening the same `WriteKudosModal` **in place** (`createPortal` to `document.body`) — the `/kudos` hero prompt bar (`WriteKudosLauncher` in `kudos-hero.tsx`), the homepage floating action button (`app/_components/widget-button.tsx`), and the FAB rules drawer's "Viết KUDOS" pill. Anonymous uses a fixed **"Ẩn danh"** sender label (`t.anonymousName`); no custom pseudonym.
